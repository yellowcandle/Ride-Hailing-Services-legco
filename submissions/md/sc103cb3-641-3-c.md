立法會CB(3)641/2026(03)號文件(修訂本)附錄A：政府中央伺服器統一跨平台累計上線時數機制

政策目標

為確保每週20小時的法定上線上限得以嚴格執行，防止司機透過切換不同網約車平台規避
時數限制，政府必須設立中央伺服器，統一累計跨平台的上線時數。

核心運作機制

步驟

說明

1. 中央伺服器設立

政府（運輸署）設立及營運中央伺服器，所有持牌網約車平台必須強制

接入。

2. 上線前驗證

每輛網約車在任一平台上線前，該平台須向中央伺服器發出驗證請求。

3. 累計時數查核

中央伺服器根據車輛許可證號碼，計算該車輛在本週（週一至週日）已

累計的總上線時數（跨平台合併計算）。

4. 批准或拒絕

• 若本週累計時數 < 20小時 → 中央伺服器回覆「批准」，平台允許該車

輛上線接單；

• 若本週累計時數 已達20小時 → 中央伺服器回覆「拒絕」，該車輛無

法於任何平台上線。

5. 實時時數更新

司機在平台上線期間，平台須實時向中央伺服器回報上線時長，確保累

計時數準確反映。

6. 每週重置

每週一零時，中央伺服器自動重置所有司機及車輛的累計時數至零。
